# LifePath Uni — Project Summary

Overview
LifePath Uni is a choice-driven, web-based simulation that helps university students (18–23) explore how everyday decisions shape long-term outcomes. The prototype emphasizes reflection over competition, showing trade-offs between academic, financial, social, and health priorities.

Objectives
- Demonstrate a multi-variable decision-making system that models time, energy, health, money, grades, and social well-being.
- Encourage reflective learning by visualizing short- and long-term consequences of routine choices.
- Deliver a playable semester-length prototype for user testing and iterative refinement.

Core Systems
- Variables: Health, Energy, Money, Grades, Social well-being.
- Scenes: Home, University, Work, Social.
- Flow: Weekly cycle (Mon–Sun) with repeated decision points; choices update variables and may branch the narrative.
- End Conditions: Semester end or critical thresholds (burnout, severe financial stress) with a final outcome summary.

Gameplay & Mechanics
- Choice-based narrative: decision nodes form a dynamic Narrative Logic Graph; branches depend on prior choices and current states.
- Feedback: status bars, event messages, and end-of-period summary showing cumulative effects and multiple possible endings (Burnout / Success / Balance).
- Replayability: emergent outcomes encourage experimentation with different strategies.

Methodology
- User-Centered Design: low-fidelity prototyping → iterative builds → usability testing with qualitative interviews.
- Data collection: track decision patterns and outcome distributions to evaluate behavioral impacts.

Technical Approach
- Web app built with React (JavaScript), HTML, and CSS for responsive UI and efficient state management.
- Simulation: state-driven model where decision nodes update variables and thresholds trigger events; narrative graph enables branching without a fully pre-authored story.
- Collaboration: GitHub for version control; Discord for coordination and project management.

Expected Outcomes
- A functional prototype that communicates trade-offs and long-term consequences.
- Increased player awareness of time, finance, and wellbeing management.
- Usability insights and quantitative data to guide future iterations.

Vision
